* Refers
* Intro
- Redundant Array of Independent Disks 独立冗余磁盘列阵
- 防止数据丢失, 提高存储性能, 把多块硬盘组合起来变成磁盘组
- RAID 磁盘出现问题, 可以通过磁盘组阵列恢复或重建文件, 比单个磁盘更可靠
* Usage
- 提高存储速度
- 提供冗余能力和提升安全性
- 提高磁盘利用率
- 平衡 CPU, 主存, 磁盘读写的差异, 提高主机综合性能

* Levels
** RAID0
- 把文件拆开装在两个硬盘上
- 读写速度 x2, 读写级别最高
- 无法恢复重建, 不提供冗余, 生成环境不用
** RAID1
- 全量双份存储, 容量 / 2
- 高冗余, 高安全, 牺牲容量和成本
** RAID5
- 检验信息在每个盘上,  A1 + A2 + A3 ... = AP, 通过校验信息加减来还原重建
- 只允许一块硬盘出故障, 大于 1 则不可重建
- 三块盘以上, 容量使用率 (N-1)/N
** RAID10
- RAID1+RAID0 镜像阵列条带
- 只能用磁盘一半的容量, 两个子阵列分别存两份一半的文件
- 优势是提高了存储速度, 故障恢复时仍可正常工作
